## Resumen

Esta hoja de ruta muestra lo que ya está disponible en Open Wearables y lo que viene a continuación. Nuestro objetivo es construir una plataforma integral que simplifique la integración de datos de wearables y habilite insights de salud impulsados por IA.

<Note>
**¿Tienes preguntas?** Únete a nuestra [comunidad de Discord](https://discord.gg/qrcfFnNE6H) para obtener ayuda, hacer preguntas y conectar con otros desarrolladores que están construyendo con Open Wearables.
</Note>

## Disponible Actualmente

Estas funcionalidades ya están disponibles:

### Portal para Desarrolladores

El portal para desarrolladores ofrece una interfaz web para gestionar tu integración con Open Wearables. Incluye:

- **Estadísticas generales**: visualiza rápidamente el número de usuarios y puntos de datos
- **Gestión de usuarios**: agrega usuarios desde el portal o mediante la API
- **Detalles del usuario**: consulta fuentes de datos conectadas, estado de integración y métricas del usuario con visualizaciones
- **Gestión de API keys**: genera y administra credenciales desde la pestaña de credenciales

---

### Gestión de Usuarios

La gestión de usuarios está disponible tanto desde el portal como desde la API REST. Esto te permite:

- Crear y administrar cuentas de usuario
- Dar seguimiento a las conexiones de usuarios con dispositivos wearables
- Monitorear el estado de sincronización de datos
- Acceder a métricas y visualizaciones de salud de cada usuario

---

### Flujo OAuth para Garmin, Polar y Suunto

Los flujos OAuth están disponibles tanto en el portal para desarrolladores como en la API REST.

La plataforma soporta autenticación basada en OAuth para proveedores wearable en la nube:

- **Garmin**
- **Polar**
- **Suunto**

Las personas usuarias pueden conectar sus dispositivos mediante un flujo simplificado:
1. Genera un enlace de conexión para tu usuario
2. La persona usuaria se autentica con su proveedor wearable
3. Los datos se sincronizan automáticamente con tu plataforma
4. Accede a ellos mediante la API unificada

---

### Sincronización de Entrenamientos y Acceso por API

**Estado**: ✅ Disponible

La sincronización de entrenamientos ya está disponible para Garmin, Polar y Suunto:

- Sincronización automática en segundo plano de datos de entrenamiento
- Formato de datos normalizado entre todos los proveedores soportados
- Acceso vía API REST a la información de entrenamientos
- Consulta de datos históricos

---

## En Desarrollo

Estas funcionalidades están en trabajo activo y estarán disponibles en próximas versiones:

### Endpoints Principales de Datos de Salud

Estamos ampliando la API para incluir endpoints de salud más completos, más allá de los entrenamientos:

- Datos de frecuencia cardiaca (reposo, activa, zonas)
- Métricas de sueño (duración, calidad, etapas)
- Resúmenes de actividad (pasos, calorías, distancia)
- Métricas corporales (peso, grasa corporal, etc.)
- Métricas de estrés y recuperación

Estos endpoints ofrecerán acceso normalizado a datos de salud entre todos los proveedores soportados, facilitando la construcción de aplicaciones de salud más completas.

---

### SDK Móvil (Flutter)

Estamos desarrollando un SDK de Flutter para simplificar la integración en apps móviles:

- **Integración con HealthKit**: permisos nativos de iOS y acceso a datos de HealthKit
- **Registro de usuario y sincronización de datos sin fricción**: proceso de onboarding que crea automáticamente usuarios en Open Wearables y sincroniza sus datos
- **Sincronización en segundo plano**: actualización automática de datos en background

Este SDK facilitará que los desarrolladores móviles integren datos de Apple Health en sus aplicaciones Flutter.

## Próximas Funcionalidades

### Automatizaciones de Health Insights

Esta será una de las funcionalidades más potentes de la plataforma: la capacidad de definir insights de salud inteligentes mediante lenguaje natural.

- **Condiciones en lenguaje natural**: describe cuándo deben activarse las notificaciones y el sistema generará condiciones bien acotadas a partir de eso
- **Notificaciones por webhook**: configura tu endpoint backend para recibir insights de salud en tiempo real
- **Prueba de automatizaciones**: ejecuta simulaciones sobre datos históricos para ver cómo funcionarían en la práctica
- **Human-in-the-loop**: marca interpretaciones incorrectas de la IA durante las pruebas para mejorar continuamente el sistema
- **Registros de automatización**: revisa activaciones pasadas y entrega retroalimentación

Esta funcionalidad permitirá construir sistemas inteligentes de monitoreo de salud sin escribir motores complejos de reglas.

<Tip>Imagina que estás desarrollando una aplicación para cuidado de personas mayores. La integración de datos de wearables puede ayudar a familiares que no pueden estar presentes a monitorear el estado de salud de sus seres queridos. Normalmente, esta integración tomaría varias semanas o meses. Con Open Wearables y el módulo de health insights, podrás implementar notificaciones por baja actividad incluso en un solo día.</Tip>

---

### Servidor MCP (Asistente de IA)

Estamos construyendo un servidor MCP (Model Context Protocol) que habilita interacciones con datos de salud impulsadas por IA. Esta funcionalidad puede ser útil tanto para desarrolladores como para usuarios finales dentro de las aplicaciones que construyas, permitiéndote integrarla directamente en interfaces de chat.

- **Integración de servidor MCP**: conecta el asistente de IA a tu chat actual o crea nuevas experiencias conversacionales
- **Consultas en lenguaje natural**: permite que las personas usuarias hagan preguntas sobre métricas de salud y entrenamientos recientes
- **Modelos de IA personalizables**: cambia de modelo según tus necesidades y caso de uso
- **Exploración de datos**: depura y explora datos de usuario mediante conversación, útil tanto para desarrollo como para soporte al usuario final

<Tip>Imagina que estás construyendo una aplicación para entrenadores personales. Gracias a Open Wearables y la integración MCP, podrás implementar fácilmente un chat donde el entrenador consulte en lenguaje natural los datos más recientes de los wearables de sus atletas.</Tip>

### Integración Mejorada de Widgets

Estamos construyendo widgets integrables que puedan añadirse fácilmente a cualquier aplicación web:

- **Widget de conexión**: permite que las personas usuarias conecten sus wearables directamente desde tu app (por ahora se las redirige a la UI de Open Wearables)
- **Widget del Asistente de Salud con IA**: integra la interfaz de chat para consultas sobre salud del usuario
- **Estilo personalizable**: adapta los widgets al sistema visual de tu aplicación

Estos widgets reducirán el tiempo de integración y ofrecerán una experiencia de usuario consistente entre distintas aplicaciones.

### Soporte de Proveedores

- **Soporte adicional de proveedores** (Fitbit, Oura, Whoop, Strava, etc.)

### Mejoras de API

- **Eventos por webhook** - Los datos de salud se envían directamente a tu backend mediante webhooks, habilitando notificaciones en tiempo real y arquitecturas orientadas a eventos

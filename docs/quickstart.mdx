## Comienza en tres pasos

Pon Open Wearables a correr localmente y empieza a integrar datos de dispositivos wearables.

### Paso 1: Prepara tu entorno

<AccordionGroup>
  <Accordion icon="copy" title="Clona el repositorio">
    Clona el repositorio de Open Wearables en tu máquina local:
    
    ```bash
    git clone https://github.com/the-momentum/open-wearables.git
    cd open-wearables
    ```
  </Accordion>
  <Accordion icon="gear" title="Configura las variables de entorno">
    Crea archivos de entorno tanto para backend como para frontend:
    
    **Configuración del backend:**
    ```bash
    cp ./backend/config/.env.example ./backend/config/.env
    ```
    
    Edita `./backend/config/.env` con tu configuración. Necesitarás definir:
    - Credenciales de base de datos
    - Configuración de Redis/Celery broker
    - Credenciales OAuth para proveedores wearable, como Garmin o Fitbit
    - Claves secretas para autenticación
    
    **Configuración del frontend:**
    ```bash
    cp ./frontend/.env.example ./frontend/.env
    ```
    
     Edita `./frontend/.env` con la configuración del frontend:
     - `VITE_API_URL` - URL de la API backend (por defecto: `http://localhost:8000`)

     <Tip>Cuando usas Docker Compose, el contenedor del frontend sobreescribe esta variable para apuntar automáticamente al backend publicado en `http://127.0.0.1:8001`.</Tip>

     <Tip>Los archivos `.env.example` incluyen credenciales por defecto para desarrollo local.</Tip>
  </Accordion>
</AccordionGroup>

### Paso 2: Inicia la aplicación

<AccordionGroup>
  <Accordion icon="docker" title="Usando Docker (recomendado)">
    La forma más simple de comenzar es con Docker Compose:
    
    ```bash
    docker compose up -d
    ```
    
    Para desarrollo con hot reload, usa el flag `--watch`:
    
    ```bash
    docker compose watch
    ```
    
    El flag `--watch` habilita sincronización automática de archivos y reinicio de servicios:
    - **Backend**: sincroniza cambios de código y reinicia servicios al actualizar archivos
    - **Frontend**: recarga en caliente cuando cambian archivos fuente
    - **Migraciones**: reinicia el backend cuando cambian archivos de migración
    - **Dependencias**: reconstruye contenedores cuando cambian archivos de paquetes
    
    Esto levantará todos los servicios necesarios:
    - Base de datos PostgreSQL
    - Redis
    - Backend FastAPI
    - Worker de Celery
    - Celery beat (scheduler)
    - Servidor de desarrollo frontend
    
     Una vez iniciado, puedes acceder a:
     - 🌐 **API**: http://127.0.0.1:8001
     - 📚 **Swagger UI**: http://127.0.0.1:8001/docs
     - 🔐 **Panel de desarrollador**: http://127.0.0.1:3000

     <Tip>Usa `docker compose logs -f` para ver los logs de todos los servicios.</Tip>
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Configuración de desarrollo local">
    Para desarrollo local sin Docker:
    
    **Configuración del backend:**
    
    1. Instala dependencias usando `uv` (recomendado) o `pip`:
       ```bash
       cd backend
       uv sync
       ```
    
    2. Configura PostgreSQL y Redis localmente
    
    3. Ejecuta las migraciones de base de datos:
       ```bash
       alembic upgrade head
       ```
    
    4. Inicia el servidor FastAPI:
       ```bash
       uvicorn app.main:api --reload
       ```
    
    **Configuración del frontend:**
    
    5. Instala dependencias del frontend usando `npm`:
       ```bash
       cd frontend
       npm install
       ```
    
    6. Inicia el servidor de desarrollo del frontend:
       ```bash
       npm run dev
       ```
    
    El frontend quedará disponible en http://localhost:3000
  </Accordion>
</AccordionGroup>

### Paso 3: Haz tu primera llamada a la API

<Accordion icon="rocket" title="Probar la API">
  Una vez que la aplicación esté corriendo, puedes probarla:

  1. **Verifica el estado del servidor**:
     ```bash
     curl http://127.0.0.1:8001/
     ```
     Debería devolver: `{"message": "Server is running!"}`

  2. **Crea una API key**:
     Al iniciar, se crea automáticamente una cuenta de administrador usando las variables de entorno `ADMIN_EMAIL` y `ADMIN_PASSWORD` (por defecto: `admin@admin.com` / `your-secure-password`).

     Usa el panel de administración en http://127.0.0.1:3000/ para iniciar sesión y generar API keys.

  3. **Carga datos de ejemplo** (opcional):
     Si quieres usuarios de prueba y datos de actividad de ejemplo:
     ```bash
     make seed
     ```

  4. **Consulta la documentación de la API**:
     Abre http://127.0.0.1:8001/docs en tu navegador para explorar la Swagger UI interactiva.

  5. **Haz solicitudes**:
      ```bash     
      # Usa la API key para hacer solicitudes autenticadas
      curl http://127.0.0.1:8001/api/v1/users \
        -H "X-Open-Wearables-API-Key: YOUR_API_KEY"
      ```
</Accordion>
